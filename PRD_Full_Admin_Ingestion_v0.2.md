# PRD: 교실 맞춤형 대시보드 포털 – 전체앱, 카테고리, 앱 런처형 (Admin Ingestion 반영)

버전: 0.2
작성일: 2026-06-27
작성자: [팀 이름 또는 담당자]
적용 범위: 교사, 학부모(읽기 전용), 학생(선택적) — 관리자 입력을 통해 앱 카탈로그를 확장하는 MVP 기능.

---

## 1) 목적 및 목표
- 교실 수업 운영의 생산성을 높이기 위해, “오늘의 수업 목표에 맞춘 전체 앱 뷰”를 제공하고, 카테고리별로 앱을 빠르게 찾을 수 있는 런처형 UI를 제공한다.
- 수업계획과 앱 매핑을 연동하여 수업 흐름을 매끄럽게 관리하고, 학생별 맞춤 추천과 과제 연동으로 학습 진행 상황을 한 눈에 파악한다.
- 교실 관리의 일관성과 투명성을 높이고, 학부모/학생도 필요 시 열람 가능한 읽기 전용 뷰를 통해 커뮤니케이션의 가치를 높인다.

주요 성공 지표
- 교사 만족도 80% 이상, 사용 의도 75% 이상
- MVP 도입 후 3개월 내 수업계획-앱 매핑 평균 시간 40% 단축
- 학생별 추천 활용률 및 과제 연동 완료율 증가

---

## 2) 문제 정의
- 다수의 앱이 분산 저장되어 있어 교사가 수업에 맞는 도구를 찾는 데 시간 소요가 큼.
- 수업계획과 학급별 학습 필요도에 따라 앱 추천이 수동적이고 비일관적임.
- 학부모/학생이 실시간으로 학습 진행 상황을 확인하기 어렵고, 피드백 루프가 분절돼 있음.
- 앱 카탈로그의 구조가 복잡하고, 특정 주제/단원에 맞춘 빠른 매핑이 어려움.

---

## 3) 대상 사용자 및 페르소나

- 주 사용자: 교사
  - 요구사항: 수업계획과 앱 매핑의 빠른 연결, 학생별 맞춤 추천, 과제 연동 및 피드백 관리
- 보조 사용자: 학부모(읽기 전용)
  - 요구사항: 자녀의 수업 계획/진도 현황 간략 확인, 안전한 접근
- 학생(선택적): 본인 학습 프로그램 접근 및 피드백 수집에 참여
  - 요구사항: 개인화된 추천 앱 목록 확인, 과제 연계 확인

권한 개요
- 교사: 전체 권한(앱 카탈로그 관리, 매핑, 추천 생성, 과제 연동)
- 학부모: 읽기 전용 대시보드 접근
- 학생: 개인화 대시보드 접근(필요 시)

## 4) 범위 (Scope)

- 포함
  - 전체앱 뷰(카테고리 기반 탐색 포함) + 앱 런처형 인터페이스
  - 수업계획 연동(앱 매핑, 목표, 소요시간 표시)
  - 학생별 맞춤 추천(진도/학습갭 반영)
  - 과제 연동 및 관리(마감일, 제출 상태, 피드백 링크)
  - 대시보드 패널 구성(오늘의 추천, 수업계획 맵, 학생 하이라이트, 과제 트래커)
  - 관리용 UI(앱 카탈로그 관리, 카테고리 관리, 권한 설정)
  - 기본 읽기 전용 학부모/학생 뷰
- 제외
  - 외부 시스템과의 실시간 싱크를 요구하는 고도화된 SIS/ERP 연동(제2단계)
  - 고도화된 분석 대시보드(예: AI 예측, 상세 학습 분석은 MVP 이후 확장)
  - 다국어 번역 자동화 전체 체계(필요 시 산출물로 간단한 다국어 태깅만 지원)

## 5) 핵심 기능 및 시나리오

### 5.1 오늘의 전체 앱 런처( launcher + 카테고리 내비게이션)
- 기능
  - 카테고리별 정리된 앱 그리드(또는 카테고리 탭)
  - 상단 검색/필터(학년, 과목, 유형, 소요시간)
  - 앱 카드에는 썸네일, 제목, 간단 설명, 학년/과목 태그, 소요시간, 매핑 버튼, 즐겨찾기
  - 앱 실행 버튼으로 새 창/새 탭에서 실행하거나 포털 내 iFrame으로 연동
- 성공 기준
  - 한 화면에서 최소 8개 카테고리, 각 카테고리당 6–12개 앱 표시 가능
  - 로딩 시간 < 1.5초(평균), 타깃 반응 시간 200ms 이내
- 수용 가정
  - 앱 엔트리는 카탈로그에 중앙 관리되며, 신규 앱 추가 시 카탈로그에 반영

### 5.2 수업계획 연동
- 기능
  - 수업 단원/주제에 맞춰 앱 매핑 드래그앤드롭 가능
  - 활동 시간, 목표, 필요한 자료를 카드/목록에 표시
- 수용 기준
  - 1) drag-and-drop으로 최소 3개의 앱 매핑 가능
  - 2) 매핑된 앱은 해당 수업 계획 패널에 시각적으로 연결 표시

### 5.3 학생별 추천 목록
- 기능
  - 각 학생의 진행도/학습갭에 따라 맞춤 앱 제안
  - 진행도 바, 주의점 표시, 빠른 실행 칩 제공
- 수용 기준
  - 학생 수 N에 대해 1:N 매핑이 가능하고, 다수의 학생에 대해 동시 업데이트 가능

### 5.4 과제 연동 및 관리
- 기능
  - 과제에 앱 사용 요구사항 연결
  - 마감일, 제출 상태, 피드백 폼 링크 제공
- 수용 기준
  - 마감일 알림 + 상태 표시는 실시간으로 갱신
  - 피드백 루프 접근성 확보

### 5.5 수업 자료 패널
- 기능
  - 수업 계획에 필요한 스크린샷/샘플 이미지, 비디오 링크, 자막/접근성 정보 표시
- 수용 기준
  - 자원 첨부 최대 5–7개까지 직관적으로 표시

### 5.6 분석/피드백 패널(MVP)
- 기능
  - 학생별 진도 트렌드, 가장 많이 사용된 앱 요약, 피드백 요약
- 수용 기준
  - 간략 요약 차트/리스트로 제공(정확한 통계 대신 샘플 뷰)

### 5.7 대시보드 내비게이션
- 기능
  - 좌측 사이드바/상단 탭으로 Home, 수업계획, 학생, 과제, 설정 간 이동
- 수용 기준
  - 2층 구조의 간단한 네비게이션으로 2초 이내 화면 전환

## 6) 데이터 모델 및 API 요구사항

### 6.1 데이터 모델(간단 스키마 아이디어)
- Class, LessonPlan, AppEntry, Category, Student, Assignment, Recommendation

### 6.2 API 엔드포인트 제안(초안)
- GET /api/apps?category={id}&grade={g}&subject={s}&query={q}
- GET /api/categories
- GET /api/classes/{classId}/lessonPlans
- POST /api/lessonPlans/{id}/mapApps
- GET /api/students/{studentId}/recommendations
- GET /api/assignments?classId={id}
- POST /api/assignments/{id}/linkApps
- GET /api/reports/class/{classId}/progress

참고: MVP 시에는 인증/권한 계층, 데이터 동기화 전략은 간략화하고, 필요 시 추후 리팩토링.

## 7) UX/UI 원칙

- 정보 구조
  - 좌측 사이드바: 홈, 수업계획, 학생, 과제, 설정
  - 메인 뷰: “오늘의 전체앱 런처”를 기본으로, 수업계획 연동 패널, 학생 추천, 과제 트래커를 모듈 형태로 배치
- 시각 디자인
  - 색상: var(--wrks-viz-primary)를 주요 포인트로 사용하되, 차분한 회색 계열 보조
  - 타이포그래피: Pretendard, 본문 14px 이상, 카드 타이틀 16-18px
  - 카드/패널: 8px 모서리, 1px 경계선, 여백 12px 이상
- 접근성
  - 충분한 대비, 키보드 네비게이션, 스크린리더 라벨 제공
  - 이미지에는 alt 텍스트, ARIA 속성 활용
- 반응형
  - 데스크탑 3–4열 카테고리 뷰, 태블릿 2열, 모바일 1열 스크롤 뷰

## 8) MVP 로드맷 및 성공 기준

- MVP 구성
  - 전체앱 뷰(카테고리 기반) + 앱 런처형 UI
  - 수업계획 연동(앱 매핑), 학생별 추천, 과제 연동 기본 구현
  - 학부모 읽기 전용 뷰 기본 제공
- 성공 기준
  - MVP의 핵심 기능이 2–3주 이내 작동 가능
  - 핵심 경로 사용 시간 KPI: 앱 런처 로딩 < 1.5초, 매핑/연동 흐름 < 3초
  - 사용자 피드백에서 70% 이상 만족도 확보
- 확장 로드맷
  - 다국어 태깅, 고도화된 분석 대시보드, 팀 협업 기능, SIS/ERP 연동은 2차 MVP에서 확장

- 성능

- 가용성
  - 99.5% 이상 가용성 목표
- 접근성
  - WCAG 2.1 AA 준수 목적의 기본 체크리스트 반영

## 11) 로드맷 / 마일스톤

- 1주차: 요구사항 확정, 데이터 모델 설계, API 스펙 확정

## 13) 용어 및 부록

- AppEntry: 포털에 등록된 개별 도구/앱의 메타데이터
- Category: 앱의 주제/용도에 따른 분류 단위
- LessonPlan: 수업의 단위(주제, 단원) 및 매핑된 앱 정보
- Recommendation: 학생 또는 수업 단위에 기반한 앱 추천 목록

부록: 예시 데이터 개요
- AppEntry 예시: { id: "app_math_frac", title: "분수의 덧셈", categoryId: "cat_math", subject: "수학", gradeLevels: [1,2], duration: "10-15분", thumbnailUrl: "...", tags: ["분수","연산"] }
- Category 예시: { id: "cat_math", name: "수학" }
- LessonPlan 예시: { id: "lp_101", classId: "cls_01", unit: "단원 3: 분수의 덧셈", topic: "피자 나누기", date: "...", duration: "2주", linkedAppIds: ["app_math_frac"] }
