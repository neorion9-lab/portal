# design.md - 교실 맞춤형 대시보드 포털 디자인 가이드 (전체앱, 카테고리, 앱 런처형)

버전: 0.1
작성일: 2026-06-27
작성자: [팀 이름 또는 디자이너]
적용 범위: 교사, 학부모(읽기 전용), 학생(선택적) — 컨셉 C에 기반한 디자인 가이드

---

## 1) 디자인 목표 및 원칙
- 목표: 교실 운영의 흐름을 돕는 직관적이며 확장 가능한 대시보드 디자인. 오늘의 수업 목표에 맞춘 전체앱 뷰와 카테고리 기반 런처를 중심으로, 수업계획 매핑과 학생 맞춤 추천을 시각적으로 통합한다.
- 원칙:
  - 간결성: 핵심 정보만 우선 표기
  - 일관성: 디자인 시스템 토큰과 컴포넌트 재사용
  - 접근성: 충분한 대비, 키보드 접근성, 스크린리더 호환
  - 반응형: 데스크탑/태블릿/모바일에서 일관된 UX
  - 비주얼의 절제: 과도한 애니메이션 제거, 1px 경계선, 단일 포인트 컬러 팔레트

---

## 2) 디자인 시스템 (Tokens)
- 색상(팔레트):
  - 기본(primary): var(--wrks-viz-primary) = #2563eb
  - 배경/표시: var(--wrks-viz-bg) = #ffffff, var(--wrks-viz-fg) = #111827
  - 보조: var(--wrks-viz-border) = #d1d5db, 회색 계열
- 타이포그래피:
  - 기본 글꼴: Pretendard, sans-serif
  - 본문: 14px
  - 제목/카드 헤더: 16-18px
  - 행간: 1.4~1.6배
- 간격:
  - 공통 패딩: 12px (space-3)
  - 섹션 간격: 8-12px
- 컴포넌트 규칙:
  - 경계선 1px, 라운드 8px, 음영 최소화
  - 아이콘은 의미 있는 경우에만 사용

---

## 3) 레이아웃 시스템
- 글로벌 레이아웃:
  - 상단 바(헤더): 로고, 검색, 알림, 프로필
  - 좌측 네비게이션(사이드바): 홈, 수업계획, 학생, 과제, 설정
  - 메인 영역: Today’s Picks, 수업계획 매핑 칸반/kanban 영역, 학생 요약, 과제 트래커, 자원 패널
- 반응형 레이아웃:
  - 데스크탑: 3-4 열 그리드의 카드 뷰, 12-column 기반
  - 태블릿: 2열 뷰
  - 모바일: 1열 스크롤 카드 뷰

---

## 4) 컴포넌트 라이브러리
- AppCard (앱 카드)
  - 속성: thumbnail, title, categoryTags, duration, mapButton, favoriteToggle
  - 상태: default / hovered / pinned / disabled
- CategoryNav (카테고리 내비게이션)
  - 수평 스크롤 또는 좌측 사이드 바형 리스트
- LauncherGrid (앱 런처 그리드)
  - 그룹화된 카테고리별 카드 컬렉션
- PlanBoard (수업계획 칸반)
  - 열: 대기, 매핑중, 매핑완료, 완료 등
  - 드래그앤드롭으로 앱 카드를 수업 계획에 매핑
- StudentPanel (학생 대시보드 패널)
  - 학생별 progressBar, 추천 앱 칩, 주의점 표시
- AssignmentRow (과제 목록 행)
  - 마감일, 상태 아이콘, 연계 앱 아이콘
- PreviewModal (미리보기 모달)
  - 앱 상세 정보, 실행 버튼, 썸네일
- AdminIngestPanel (관리자 인제스션 UI)
  - 새 앱 추가 폼 구성 요소
- 기타: Skeleton, EmptyState, Tooltip, Badge, Drawer, Modal

---

## 5) 상호작용 패턴
- Drag-and-Drop: 수업계획에 앱 매핑
- 즉시 미리보기: 앱 카드에서 미리보기 모달 열기
- 검색+필터: 학년/과목/유형/소요시간 조합 기반 탐색
- 런처 실행: 선택한 앱을 새 탭 또는 iFrame으로 열기
- 상태 표시: 예) 매핑 상태, 과제 상태, 진행도 상태

---

## 6) 접근성 가이드
- WCAG 2.1 AA 준수 목표
  - 충분한 대비(4.5:1 이상) 
  - 키보드 탐색 순서 명확
  - 이미지 alt, aria-label 사용
  - 스크린리더 친화적 구조(semantic HTML + ARIA)

---

## 7) 시각 자료/그래픽 가이드
- 앱 이미지/섬네일
  - 권장 해상도: 512x512px, 최대 2MB
  - 비율 유지: 정사각형 또는 4:3 비율 권장
- 아이콘/일러스트
  - 간결하고 직관적인 아이콘 세트 사용
- 애니메이션
  - 최소화, hover 정도의 미세한 트랜지션(150ms)

## 8) 개발자 핸드오프까지의 산출물 제시

## 9) QA 체크리스트
- 시나리오: 홈 > 수업계획 > 앱 매핑 > 저장 흐름 정상 동작
- 반응형 체크: 3가지 뷰 포맷에서 레이아웃 적합성
- 접근성 검사: 키보드, 스크린리더, 색상 대비 확인
- 퍼포먼스: 로딩 시간 및 간단한 애니메이션 총합

---

## 10) 로드맵 연계
- 1차: MVP 대상 컴포넌트 라이브러리 확정 및 디자인 시스템 정착
- 2차: Admin Ingestion UI 및 인제스트 워크플로우 통합
